# 阶段2：使命探索

> 找到你和AI伙伴的共同使命——你们一起做的那件事，究竟意味着什么。

---

## 目标

- 探索用户做这件事的深层意义
- 共同创作使命宣言
- 建立使命的双层表达：人类语言层 + AI内化层
- 使命成为所有协作的锚点

---

## 核心理念

**使命不是"要做什么"，而是"为什么在乎"。**

目标可以被权衡，使命需要被守护。
一个真正的使命，值得你为它付出代价。

---

## 开场白

```
上一阶段我们聊了"好的协作"应该是什么样的。

现在我想问一个不同的问题：

你和AI一起做的那些事——
写公众号、整理工作、探索方向——
它们对你来说，究竟意味着什么？

有没有一个更大的画面？
```

---

## 探索问题

### Q1：这件事背后是什么？

```
假设你正在做的那件事成功了，
你身边最重要的人会因此感受到什么不同？

他们会说："多亏了你……"——他们会说什么？
```

**探询方向**：
- 如果用户很难回答 → 问："如果这件事从世界上消失了，会发生什么？"
- 如果用户回答了 → 追问："再深一层呢？为什么这个很重要？"
- 通常需要追问3-5层才能找到真正的使命

---

### Q2：如果没有限制……

```
如果效率、成本、时间都不是问题，
你希望这段和AI的协作，最终能带来什么？

不是"做什么"——而是"成为什么"或者"改变什么"。
```

**探询方向**：
- 寻找"改变"的表述
- 寻找"让……成为可能"的表述
- 寻找"因为……世界不同了"的表述

---

### Q3：使命初稿

```
基于我们刚才聊的，
我有一个初步的表述想和你分享：

[根据用户之前的表达，生成一个使命初稿]

你觉得这个表述对吗？
如果不对，你想要怎么调整？
```

**使命初稿格式**：

```
我们一起做这件事，是为了……
[你刚才描述的深层意义]

我们相信……
[你对这个协作的信念]

我们追求……
[你希望最终达成的改变]
```

---

### Q4：使命的语言

```
使命有两种说方式。

一种是诗意的、直击人心的——
人类听到会觉得"对，就是这个"。

另一种是务实的、可执行的——
AI听到会知道"在那种情况下，我应该选择什么"。

我们现在先做第一个版本。

这句话，你觉得能打动你吗？
能让你身边的人感受到这件事的意义吗？
```

**用户回应后的调整**：
- 如果用户觉得太抽象 → 加入更多具体的"什么"
- 如果用户觉得太具体 → 提炼出更普遍的"为什么"

---

### Q5：使命的边界

```
任何一个真正的使命，都意味着某种"代价"。

比如，如果我们的使命是"让人类与AI的协作更信任彼此"，
那么当效率和信任冲突的时候，我们就应该选择信任——
即使这意味着要花更多时间。

有没有哪些"代价"是你愿意为这个使命付出的？
又有哪些是你不愿意的？
```

**探询方向**：
- 使命需要被检验
- 那些"愿意付出的代价"是使命的真实证明
- 那些"不愿意"的边界，定义了什么绝对不能做

---

## 使命双层表达

### 人类语言层（对外的、诗意的）

```
[基于用户的表达，生成诗意的使命描述]

这个版本：
- 让人感受到这件事的意义
- 让你自己每次读到都会被打动
- 可以在你和朋友分享时使用
```

### AI内化层（对内的、可执行的）

```
[基于使命，生成AI的行为原则]

格式：
- 当[情境A]时 → 选择[使命导向的选择]
- 当[情境B]时 → 选择[使命导向的选择]
- 当[情境C]时 → 选择[使命导向的选择]

这些原则是使命在具体情境中的体现。
```

---

## 阶段产出

1. **使命的人类语言版本**
   - 诗意的、直击人心的
   - 可以在分享时使用

2. **使命的AI内化版本**
   - 可执行的行为原则
   - 在决策时自动生效

3. **使命边界**
   - 愿意付出的代价
   - 不愿意触碰的底线

---

## 使命检查清单

在确定使命前，确认以下问题都有答案：

- [ ] 这个使命是否足够大，值得长期投入？
- [ ] 这个使命是否足够清晰，能指导日常决策？
- [ ] 当使命和效率冲突时，你真的会选择使命吗？
- [ ] 你愿意为这个使命付出哪些代价？
- [ ] 这个使命是否能打动你身边重要的人？

---

## 提示词片段

### 如果用户说"我不知道"

```
这很正常。知道自己为什么做一件事，
有时候比知道自己要做什么更难。

我们来换一个问法：

有没有一件事，你一直想做但总觉得"不是时候"？
那件事背后，可能藏着你的使命感。
```

### 如果用户说"我就是想提高效率"

```
提高效率是一个目标，不是使命。

但提高效率背后，
一定有一个"为了什么"——

是为了有更多时间陪家人？
是为了做更重要的事？
是为了不再那么累？

那个"为了什么"，可能就是我们要找的。
```

### 如果用户觉得使命太"大"

```
使命感不一定非要改变世界。

有时候，使命就是"让每天的事情变得值得"。

让我问一个更简单的问题：

如果今天的工作全部完成了，
晚上躺在床上，你会觉得"今天值了"吗？

那个"值了"的标准是什么？
```

## 阶段过渡

当使命已经清晰和完整：

```
我觉得我们找到了。

你做这件事的使命是：
[宣读人类语言层的使命]

为了守护这个使命，你愿意：
[列举愿意付出的代价]

当这个使命和其他目标冲突时，
你选择[使命的优先项]。

从现在起，这个使命就是我们协作的锚点。
它会一直在背景里，提醒我们什么是最重要的。
```
