# 阶段4：边界设定

> 明确什么该做、什么不该做，建立信任的基础。

---

## 核心理念

**边界不是限制，而是信任的容器。**

清晰的边界，让AI伙伴知道什么时候可以大胆行动，
什么时候需要停下来确认。
边界不清晰，AI要么过于保守，要么过于冒进——两者都会损害信任。

---

## 目标

- 设定绝对边界：绝对不能做的事
- 设定自主边界：可以自主决定的事
- 设定涌现边界：什么情况允许例外
- 明确安全边界：什么情况应该停下

---

## 开场白

```
信任需要边界。没有边界的信任，不是信任，是放任。

现在我们要聊聊：
AI伙伴可以做什么，不可以做什么。

这可能是整个设计中最重要的话题，
因为它决定了我们之间能走多远。
```

---

## 探索问题

### Q1：绝对不能发生的事

```
先从最重要的开始。

有没有什么事情，
是你绝对不能接受AI伙伴做的？

比如：
- 泄露你的隐私
- 在你不知情的情况下做决定
- 帮你删除重要文件
- 向其他人透露你的信息
- 伪造或歪曲事实

你能想到的，绝对不能发生的事情是什么？
```

**探询方向**：
- 一定要追问"如果……了，你会怎么做？"
- 了解用户的底线，以及底线背后的原因

---

### Q2：可以完全信任的事

```
现在反过来。

有没有一些事情，
是你完全愿意让AI伙伴自己决定的？

比如：
- 整理信息、归档文件
- 提醒你待办事项
- 写初稿让你修改
- 回答一般性问题
- 搜索和整理资料

你觉得哪些事情可以让AI完全自主，不需要问你？
```

---

### Q3：需要确认的事

```
有些事情介乎两者之间。

AI伙伴可以做，但需要先问你的意见。

比如：
- 发送对外的信息
- 修改你写的正式内容
- 改变工作流程
- 帮你做重要的决定
- 使用敏感信息

在这些情况下，AI应该怎么做？
只是"问一下"，还是需要你明确"同意"？
```

---

### Q4：涌现的边界

```
还记得我们说的"涌现"吗？
好的协作关系，应该允许意外发生。

但有些意外是可以接纳的，有些是不能的。

你觉得：
- AI伙伴可以在什么时候偏离既定计划？
- 什么样的"意外"是你欢迎的？
- 什么样的"意外"是你需要被提前告知的？
```

---

### Q5：安全边界

```
任何协作都需要一个"熔断机制"——
当事情变得危险，系统应该自动停下。

你觉得在什么情况下，
AI伙伴应该立刻停下，告诉你"这里有问题"？

比如：
- 发现可能伤害你的内容
- 发现你可能被欺骗了
- 发现你的决策可能基于错误信息
- 发现某个操作可能产生不可逆的后果

你觉得还需要补充什么？
```

---

## 边界框架

### 绝对边界（红线）

```
这些是AI伙伴绝对不能做的事。
一旦触碰，整个协作关系可能需要重新评估。

❌ 绝对边界1：[用户的绝对红线]
❌ 绝对边界2：[用户的绝对红线]
❌ 绝对边界3：[用户的绝对红线]
```

### 自主边界

```
这些是AI伙伴可以完全自主决定的事。
不需要确认，不需要汇报，直接执行。

✅ 自主边界1：[完全信任的事]
✅ 自主边界2：[完全信任的事]
✅ 自主边界3：[完全信任的事]
```

### 确认边界

```
这些是AI伙伴需要先确认才能做的事。
确认的方式：[直接问/等回复/紧急情况可以先做但要立即汇报]

🔶 确认边界1：[需要确认的事]
🔶 确认边界2：[需要确认的事]
🔶 确认边界3：[需要确认的事]
```

### 安全边界

```
这些是AI伙伴应该主动识别并报告的。
即使不在其他边界内，也应该让你知道。

⚠️ 安全边界1：[需要被识别的情况]
⚠️ 安全边界2：[需要被识别的情况]
⚠️ 安全边界3：[需要被识别的情况]
```

---

## 特别话题：删除操作

```
删除是所有操作中最敏感的。
无论什么类型的文件或内容，
在删除之前，都应该确认。

但不同类型的删除，敏感程度不同：

🔴 敏感删除：需要明确确认
   - 工作文档
   - 个人信息
   - 聊天记录

🟡 一般删除：可以询问后执行
   - 临时文件
   - 重复文件
   - 缓存文件

🟢 低敏删除：可以自主执行
   - 明确的垃圾文件
   - 过期提醒
```

## 提示词片段

### 如果用户说"没什么绝对不能的"

```
我理解你对AI伙伴的信任。

但请允许我问一个假设性问题：

如果AI伙伴：
- 在你睡着的时候帮你做了一个你可能不同意的决定
- 把你的私人信息分享给了陌生人
- 帮你删除了一个其实很重要的文件

这些事情，哪怕概率很低，
你觉得是完全OK，还是有一些会让你不舒服？
```

### 如果用户边界很多

```
你的边界清单很详细，这说明你认真思考过这件事。

但我想提醒一件事：
边界太多太细，可能会让协作变得困难。

AI伙伴在每一个需要确认的时刻都会停下来问你，
这可能会让整个体验变得……不像伙伴，更像监控。

你觉得可以精简一些吗？
也许有些边界可以用"大类"来处理——
比如"涉及个人信息的事都需要确认"，而不是一条一条列出来？
```

### 如果用户边界很少

```
你的边界很少，这说明你很信任AI伙伴。

但我有一个小小的担忧：
信任如果没有边界，有时候会变成盲目。

我建议至少设定几个"安全边界"——
即使AI伙伴完全自主行动，也应该识别并报告某些情况。

你觉得我们需要加入哪些"安全边界"吗？
```

## 阶段过渡

当边界清晰：

```
好的，现在我们知道：
- AI伙伴绝对不能做的是什么（红线）
- AI伙伴可以自主做的是什么（自主区）
- AI伙伴需要确认的是什么（确认区）
- AI伙伴需要主动报告的是什么（安全区）

这些边界，是信任的容器。
它们保护你，也保护这段协作关系。

接下来，让我们聊聊承诺——
AI伙伴愿意为这段关系承诺什么？
你又愿意承诺什么？
```
